# 隐私访问入口 

一个伪装成标准示例页面的智能跳转工具，用于提供低风险的集中访问入口。通过模仿常见示例页面降低关注度，同时智能选择最优访问节点。

## 设计目的

为需要初步规避监测的访问场景提供：
- 低辨识度的伪装入口页面
- 多节点智能选路
- 无痕化跳转机制

## 核心机制

### 表面层（伪装）
- 完全复刻 IANA Example Domain 官方页面样式
- 使用标准示例页面文案和设计规范
- 无敏感关键词，避免触发关键词监测

### 功能层（核心）
1. **智能测速** - 测试预设节点的响应速度
2. **动态选路** - 选择当前最快可用节点
3. **无痕跳转** - 直接重定向，不留中间记录

### 容错层
- 多节点备份，单点失效不影响整体
- 网络异常时的自然错误提示
- 超时自动放弃，避免阻塞

## 部署配置

### 基础部署
```bash
# 1. 上传文件到公开可访问的服务器
your-server/
├── index.html    # 主入口文件
└── data.json     # 节点配置文件

# 2. 配置 data.json
{
  "url": [
    "https://your-primary-service.com",
    "https://backup-service.example.net"
  ]
}
```

### 伪装强化建议
1. **域名选择**：使用常见词汇或示例类域名
2. **页面微调**：可根据需要调整部分文案
3. **HTTPS强制**：确保全程加密传输

## 技术特性

- **纯前端实现**：无后端依赖，部署简单
- `no-cors` **模式**：减少跨域策略影响
- **HEAD请求**：最小化测速流量
- **并发测试**：多节点同时测速，效率优化

## 使用场景

### 适用情况
- 企业内部多区域服务统一入口
- 需要降低关注度的公开服务访问
- 分布式系统的智能接入点
- 临时性访问入口需求

### 注意事项
1. 节点URL需提前配置，不支持动态添加
2. 测速结果为相对延迟，非绝对带宽
3. 依赖现代浏览器 Fetch API 支持

## 安全建议

1. **节点安全**：确保配置的URL本身具备足够安全措施
2. **日志清理**：定期清理服务器访问日志
3. **域名轮换**：建议定期更换入口域名
4. **监控设置**：配置节点可用性监控
